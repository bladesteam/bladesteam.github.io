4.3. Client

class client.BladesClient(client_id: Optional[str] = None, device: Union[torch.device, str] = 'cpu')[source]

Bases: object

Base class for all clients.

Note

Your honest clients should also subclass this class.

Parameters

  • client_id (str, optional.) – a unique id of the client.

  • device (torch.device, or str) – if specified, all parameters will be copied to that device

set_model(model: torch.nn.modules.module.Module, opt: type, lr: float) → None[source]

Deep copy the given model to the client.

Note

To improve the scalability, this API may be removed in the future,

Parameters

  • model (torch.nn.Module) – a Torch model for current client.

  • opt (torch.optim.Optimizer) – client optimizer

  • lr (float) – local learning rate

local_training(num_rounds: int, use_actor: bool, data_batches: list) → None[source]

Local optimizaiton of the client. Byzantine clients can overwrite this method to perform adversarial attack.

Parameters

  • num_rounds (int) – Number of local optimization steps.

  • use_actor (bool) – Specifying the training mode, it should be True if you use Trainer Mode

  • data_batches (list) – A list of training batches for local training.

get_update() → torch.Tensor[source]

Return the saved update of local optimization, represented as a vector.

class client.ByzantineClient(client_id: Optional[str] = None, device: Union[torch.device, str] = 'cpu')[source]

Bases: client.BladesClient

Base class for Byzantine clients.

Note

Your Byzantine clients should also subclass this class, and overwrite local_training and omniscient_callback to customize your attack.

omniscient_callback(simulator)[source]

A method that will be registered by the simulator and execute after each communication round. It allows a Byzantine client has full knowledge of the training system, e.g., updates from all clients. Your Byzantine client can overwrite this method to access information from the server and other clients.

Parameters

simulator (Simulator) – The running simulator.
