Simulator

class simulator.Simulator(dataset: blades.datasets.datasets.FLDataset, aggregator: Union[Callable[[list], torch.Tensor], str], num_byzantine: Optional[int] = 0, attack: Optional[str] = 'None', num_actors: Optional[int] = 4, mode: Optional[str] = 'actor', log_path: str = './outputs', metrics: Optional[dict] = None, use_cuda: Optional[bool] = False, **kwargs)[source]

Synchronous and parallel training with specified aggregators.

Parameters

  • dataset – FLDataset that consists local data of all clients

  • aggregator – String (name of build-in aggregation scheme) or a callable which takes a list of tensors and returns an aggregated tensor.

  • num_byzantine (int, optional) – Number of Byzantine clients under build-in attack. It should be 0 if you have custom attack strategy.

  • attack (str) – None by default. One of the build-in attacks, i.e., None, noise, labelflipping, signflipping, alie, ipm. It should be None if you have custom attack strategy.

  • num_actors (int) – Number of Ray actors that will be created for local training.

  • mode (str) – Training mode, either actor or trainer. For large scale client population, actor mode is favorable

  • log_path (str) – The path of logging

run(model: torch.nn.modules.module.Module, server_optimizer: Union[torch.optim.optimizer.Optimizer, str] = 'SGD', client_optimizer: Union[torch.optim.optimizer.Optimizer, str] = 'SGD', loss: Optional[str] = 'crossentropy', global_rounds: Optional[int] = 1, local_steps: Optional[int] = 1, validate_interval: Optional[int] = 1, test_batch_size: Optional[int] = 64, lr: Optional[float] = 0.1, lr_scheduler=None)[source]

Run the adversarial training.

Parameters

  • model (torch.nn.Module) – The global model for training.

  • server_optimizer (torch.optim.Optimizer or str) – Pytorch optimizer for server-side optimization. Currently, the str type only supports SGD

  • client_optimizer (torch.optim.Optimizer or str) – Pytorch optimizer for client-side optimization. Currently, the str type only supports SGD

  • loss (str) – A Pytorch Loss function. See https://pytorch.org/docs/stable/nn.html#loss-functions. Currently, the str type only supports crossentropy

  • global_rounds (int, optional) – Number of communication rounds in total.

  • local_steps (int, optional) – Number of local steps of each global round.

  • validate_interval (int, optional) – Interval of evaluating the global model using test set.

  • test_batch_size (int, optional) – Batch size of evaluation

  • lr (float, optional) – Learning rate of client_optimizer

Returns

None
